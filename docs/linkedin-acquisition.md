# LinkedIn Client Acquisition — SOP

## Strategy
LinkedIn is the primary outbound channel for Sonar's own client acquisition.
The ICP is active on LinkedIn. Tiago has credibility and context there. It's lower friction than cold email for a $10K+ deal where the relationship matters.

Expand.io (or Phantom Buster) automates the volume. Claude handles the personalization. Tiago handles the conversations and closes.

## The workflow

### Step 1 — Build the prospect list (Tiago)
Source: LinkedIn Sales Navigator (Tiago's access)
Search criteria:
- Title: Founder, CEO, CRO, VP Sales, Head of Revenue, Head of Growth
- Company size: 20-150 employees
- Industry: SaaS, tech-enabled services
- Geography: US, Canada, UK (English-speaking markets)
- Recent activity: active on LinkedIn in the last 30 days

Output: list with name, company, LinkedIn URL
Format: CSV or pasted bullet list — whatever is easiest for Tiago to produce

Target: 50-100 new prospects per week to feed the workflow

### Step 2 — Research + score (Claude)
For each prospect on the list, run the linkedin-outreach skill.
This triggers research-account for each account and scores them.
Prospects below 50 ICP score → flagged, not included in outreach batch.

### Step 3 — Generate DM sequences (Claude)
For each approved prospect (score 50+), run draft-linkedin-dm.
Output: a batch review document with all DM sequences.

### Step 4 — Tiago review (Tiago)
Tiago reads the batch document. For each sequence:
- Approve → goes into Expand.io
- Reject → note why (helps improve future batches)
- Edit → Tiago edits inline, then approves

Aim: Tiago reviews a batch of 20-30 prospects in 15-20 minutes. If review is taking longer, sequences are too generic — fix the research step.

### Step 5 — Load to Expand.io (Lucas or Tiago)
Load approved sequences into Expand.io.
Expand.io configuration:
- Tiago's LinkedIn account connected
- Pre-approval: ON (Tiago confirms each action before it fires, until volume is validated)
- Daily limits: 20-30 profile visits + 10-15 connection requests (stay within LinkedIn's safe limits)
- Sending window: Mon-Fri, 9am-5pm Tiago's timezone

### Step 6 — Conversation handling (Tiago)
When a connection accepts and a DM sequence starts:
- Replies go to Tiago's LinkedIn inbox
- Hot reply (positive, wants to talk): Tiago books a call immediately
- Warm reply (curious, not ready): Tiago continues the conversation manually
- Cold reply (not interested): Tiago removes from sequence

Tiago owns all conversations from the first reply onward.

### Step 7 — Track results (weekly)
Run weekly-report skill on Fridays.
LinkedIn metrics to track:
- Connection requests sent
- Connections accepted (acceptance rate target: 20-35%)
- DMs sent (after connection)
- Replies received (reply rate target: 5-15%)
- Discovery calls booked from LinkedIn (weekly target: 2-5)

## LinkedIn content as acquisition support
Tiago's LinkedIn posts (generated by content/tiago-post skill) warm the audience.
Before sending a connection request, Expand.io visits the profile (awareness).
Tiago's recent posts on the target's feed create context that makes the DM less cold.
This is the "content flywheel" — posts generate awareness, DMs convert to conversations.

## Safety notes
Never blast 100+ connection requests in a day — LinkedIn will flag the account.
Always use the pre-approval gate at the start of any new campaign type.
Monitor LinkedIn's usage warnings in Tiago's account settings.
If account gets restricted: pause all automation immediately, do manual outreach for 2 weeks, then restart at lower volume.
